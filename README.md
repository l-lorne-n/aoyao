# Aoyao 轻量病历语音录入工具

这是一个面向诊所场景的本地轻量病历录入 MVP。目标不是建设完整诊所管理系统，而是先解决纸质病历里自由文本书写负担重的问题：医生在病历页面中点击录音，口述内容经腾讯云语音识别转成文字，再由医生人工确认、追加或替换到对应字段。

## 当前功能

- 独立语音识别测试页：用于先验证真实口述识别效果。
- 病历录入页：围绕建档信息、既往史、过敏史、诊断记录和备注组织。
- 本地 SQLite 数据库：保存病历记录，不依赖远程数据库。
- 病历新建、保存、搜索、打开修改、软删除、垃圾桶恢复和彻底删除。
- 支持地址分类和病历编号组合管理：同一地址内编号唯一，不同地址可复用同一编号；空地址按“无地址”单独成组。
- 左侧检索拆分为“地址、编号”和“姓名、电话、诊断”两个搜索框，同时填写时取交集。
- 支持按时间查看历史病历：同一天的首次建档和诊断记录会汇总成小卡片列表。
- 既往史、过敏史、诊断记录、内调方案、备注等自由文本字段支持语音录入。
- 地址、编号、年龄和诊断指标支持直填语音：识别后直接写入字段，不弹出追加/替换确认窗。
- 诊断记录默认 4 次，可继续增加第 5 次、第 6 次等。
- 每次诊断记录包含“辨证”“内调方案”“时间”和五个小指标：血压、心率、血糖、尿酸、夜尿。
- 第一次诊断记录时间默认跟建档时间一致，后续可单独修改。
- 血压支持两个数字自动整理为 `120/80` 格式；其它诊断指标会清理为数字。
- 语音识别结果确认浮窗做了放大处理，便于视力不好的使用者阅读和确认。
- 表单修改后顶部保存状态栏变淡黄色，提示“有未保存修改”；保存后恢复白色。
- 关闭或刷新页面、切换病历、新建病历前会提示未保存修改，避免误覆盖。
- 支持从左侧按钮批量选择病历并导出纸质病历本风格 PDF。
- 支持手柄辅助录入：十字键/左摇杆选择控件，右摇杆滚动页面，A 点击/确认，Y 录音，X 保存，B 从文本框末尾删除。
- 手柄保存后会尽量保持在最近一次编辑的输入框，避免跳回左侧“新建病历”。
- 地址、性别、日期等选择控件支持手柄打开选择层并确认。
- 刷新病历页时默认打开最近编辑病历；点击“新建病历”不会自动填写病历编号。
- 腾讯云密钥只在本地 Python 后端读取，不暴露到前端页面。

## 技术栈

- 后端：Python 标准库 HTTP 服务
- 数据库：SQLite
- 前端：原生 HTML/CSS/JavaScript
- 语音识别：腾讯云一句话识别，默认引擎 `16k_zh_medical`
- PDF 导出：`reportlab`

当前实现刻意保持少依赖，方便在普通轻薄本上直接运行。

## 配置密钥

推荐复制 `.env.example` 为 `.env` 后填写：

```env
TENCENT_SECRET_ID=
TENCENT_SECRET_KEY=
TENCENT_REGION=ap-guangzhou
TENCENT_ASR_ENGINE=16k_zh_medical
TENCENT_ASR_TRANSPORT=auto
```

也兼容根目录下的密钥文件：

```text
tecent_api_key.txt
```

格式：

```text
SecretId:你的 SecretId
SecretKey:你的 SecretKey
```

真实 `.env`、密钥文件、数据库、录音样本和 PDF 导出文件均已加入 `.gitignore`，不要提交。

## 启动

在项目根目录运行：

```powershell
python -m pip install -r requirements.txt
python .\asr_test_page\server.py --host 127.0.0.1 --port 8765
```

打开病历录入页：

```text
http://127.0.0.1:8765/records.html
```

打开语音识别测试页：

```text
http://127.0.0.1:8765/
```

## 数据保存策略

- 所有病历保存在本地 SQLite：`data/aoyao_records.sqlite3`
- 页面不会自动保存，必须点击“保存病历”或用手柄 X 键保存才会写入数据库。
- 新病历保存时插入新记录；已有病历保存时更新原记录。
- 数据库内部 `id` 只给程序使用；页面上的“病历编号”是可手动配置的业务编号。
- 非空病历编号在同一地址内必须唯一；保存时如果重复，会提示已被哪条病历占用。
- 不同地址可以使用相同编号；未填写地址的病历归入“无地址”编号组。
- 主页面删除会先进入垃圾桶，不会立即物理删除。
- 垃圾桶内可复原或彻底删除；复原时如果同地址同编号已被当前病历本占用，会阻止复原并提示原因。
- 数据库中单独保存常用检索字段，同时完整病历 JSON 保存在 `data_json` 字段，便于保留动态诊断次数和指标。
- 关闭或刷新页面前使用浏览器原生离开提示；切换病历、新建病历、页面内跳转使用自定义未保存确认窗。

## 已完成工作总结

1. 梳理并确认 MVP 范围：只做语音转文字和病历录入，不做自动诊断、不做治疗方案生成。
2. 接入腾讯云一句话识别 HTTP 调用，并保留官方 SDK 可选通道。
3. 完成独立录音识别测试页，支持浏览器录音、16k WAV 转换、识别结果显示和复制。
4. 搭建病历录入页面，字段结构参照真实使用场景持续调整。
5. 增加 SQLite 本地数据库和病历 CRUD API。
6. 支持自由文本字段语音录入，识别结果可追加、替换或取消。
7. 支持地址分类、手动病历编号、同地址编号唯一校验和组合检索。
8. 支持动态增加诊断记录次数，每次诊断记录包含辨证、内调方案、时间和五项指标。
9. 去除当前不再使用的主诉、体征表现、月经、舌脉象、调摄建议和回访情况模块。
10. 增加未保存状态提示和离开保护，降低修改后忘记保存或误覆盖的风险。
11. 增加批量选择病历并导出纸质病历本风格 PDF。
12. 增加按日期浏览历史病历的时间线页面，可从小卡片跳回完整病历。
13. 增加手柄导航、手柄录音、手柄保存、手柄删除和选择层操作。
14. 优化手柄导航滚动策略，减少诊断记录内部上下移动时的突兀翻页。
15. 优化手柄保存后的焦点恢复，尽量保持在最近一次编辑的输入框。
16. 增加垃圾桶软删除、复原和彻底删除。
17. 配置 `.gitignore`，避免提交密钥、数据库、录音样本、PDF 导出文件和患者信息。
